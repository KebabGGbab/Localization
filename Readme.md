# Localization

The library is designed to simplify the process of localizing applications, primarily desktop ones. Currently, the library supports changing the current UI language culture without restarting the application, a localization service (the current thread by default), support for multiple localization providers, and markup extensions for the WPF and AvaloniaUI frameworks.

---

## Get Started

Localization can be installed using the NuGet package manager or the .NET CLI.

```
Install-Package KebabGGbab.Localization
Install-Package KebabGGbab.Localization.WPF
Install-Package KebabGGbab.Localization.AvaloniaUI
```

## Example

### Samples

There are [short samples](https://github.com/KebabGGbab/Localization/tree/main/samples) to run localization on supported frameworks.

### Configure

If you use dependency injection, you can write your own extension to IServiceCollection by calling IServiceCollection.AddLocalization(), optionally passing the initial culture and localization service, and add a localization provider, such as the Resx file provider implemented in the library and added by calling IServiceCollection.AddResxLocalization(), passing the ResourceManager generated by the ResXFileCodeGenerator tool, as well as all supported cultures. For example:

```csharp
    public static class LocalizationExtensions
    {
        public static IServiceCollection AddLocalizationWithResx(this IServiceCollection services)
        {
            ArgumentNullException.ThrowIfNull(services);

            services.AddLocalization(); // or services.AddLocalizationWPF() if you are developing a WPF application.
            services.AddResxLocalization(StringsUI.ResourceManager, [CultureInfo.GetCultureInfo("ru-RU"), CultureInfo.GetCultureInfo("en-US")]);

            return services;
        }
    }
```

### xaml

Once the library is configured, you can use it in XAML. To do this, add the namespace:

- in WPF:

```xaml
xmlns:l="clr-namespace:KebabGGbab.Localization.WPF;assembly=KebabGGbab.Localization.WPF"
```

- in AvaloniaUI:

```xaml
xmlns:l="https://github.com/KebabGGbab"
```

and use it like this:

```xaml
Title="{l:Localization MainWindowTitle}"
```

where MainWindowTitle is the resource key.
